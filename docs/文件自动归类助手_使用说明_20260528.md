# 文件自动归类助手使用说明

## 软件用途

这个工具用于整理退税材料。把待整理的 PDF、图片等文件放进监控文件夹后，软件会按公司主台账里的发票号归类到输出文件夹，并判断每票材料是否齐套。

软件会识别这些材料：

- 外销
- 报关委托书
- 报关费发票
- 提单
- 装箱单
- 运费发票
- 采购合同

其中采购合同不是必需材料；其余材料用于齐套判断和闭环校验。

## 文件名直接通过规则

除采购合同外，如果文件名是“当前发票号 + 材料标准名”，软件会直接按对应材料识别，不再等待 OCR 读出正文。

当前支持两种写法：

```text
LY25112112装箱单.jpg
25112112装箱单.jpg
```

同样适用于外销、报关委托书、报关费发票、提单、装箱单、运费发票，例如：

```text
25112112外销.pdf
25112112报关委托书.pdf
25112112报关费发票.pdf
25112112提单.pdf
25112112装箱单.jpg
25112112运费发票.pdf
```

文件名前面的数字必须能对应当前票的发票号。`25112113装箱单.jpg` 放到 `LY25112112` 这一票下时，不会按这条规则自动通过。

材料名后面不要再加其他说明。`25112112装箱单.jpg` 可以直接通过，`25112112装箱单补充.jpg` 仍会进入复核。

采购合同不使用这条规则。采购合同通常来自扫描仪软件导出的统一格式，仍按原有采购合同文件名规则识别。

## 第一次使用前的文件摆放

建议把程序放在一个单独文件夹里，例如 `自动归档`。

最小可用结构：

```text
自动归档\
  contract-router.exe
  data\
    公司系统出运统计发票号清单.xlsx
```

如果需要让闭环判断同时核对出运日期、目的国和金额，可以在同一个 `data` 文件夹里再放一个可选文件：

```text
自动归档\
  contract-router.exe
  data\
    公司系统出运统计发票号清单.xlsx
    出运闭环台账.xlsx
```

`app_settings.json` 和 `materials_config.json` 不需要提前准备。第一次打开软件时会自动生成。

如果电脑里已经有旧版本留下的 `app_settings.json`，建议先删除旧文件，再打开新版 `contract-router.exe`。旧配置可能记录了以前的主台账路径，导致主台账明明放在 `data` 目录里，软件仍然提示未找到。

## 台账要求

主台账文件必须放在 exe 同级的 `data` 文件夹里：

```text
自动归档\data\公司系统出运统计发票号清单.xlsx
```

文件名必须完全一致：

```text
公司系统出运统计发票号清单.xlsx
```

Windows 默认会隐藏文件扩展名。检查文件名时，先在资源管理器里打开：

```text
查看 -> 文件扩展名
```

确认真实文件名不是下面这些形式：

```text
公司系统出运统计发票号清单.xlsx.xlsx
公司系统出运统计发票号清单.xls
公司系统出运统计发票号清单.xlsx.xls
```

台账第一行需要有 `发票号` 这一列。软件还会读取这些列用于闭环校验：

- `出运日期`
- `目的国(地区)` 或 `目的国`
- `金额`

这些列缺失时，软件仍可按发票号归类，但对应字段会在闭环结果里显示为未核验。

`公司系统出运统计发票号清单.xlsx` 是主台账。软件只处理这里存在的发票号；其他 Excel 里多出来的发票号不会自动进入本批整理范围。

`出运闭环台账.xlsx` 是可选补充台账，适合放从出运或会计系统拉出的明细。它可以有多个 sheet，例如 `1月`、`2月`、`3月`、`4月`。第一行需要包含：

- `发票号`
- `出运日期`
- `目的国(地区)` 或 `目的国`
- `金额`

软件读取补充台账时，只会给主台账里已有的发票号补 `出运日期`、`目的国`、`金额`。补充台账里多出来的发票号会被忽略，不会生成新文件夹，也不会参与财务批次判断。

如果主台账和补充台账同一票的字段不一致，软件不会覆盖主台账原值，会在 `判断报告` 或 `批次统计报告` 里写入 `台账字段冲突`，该票需要人工复核。

## 工作文件夹不要和程序放在一起

不要把待整理材料直接放进 `自动归档` 文件夹。否则软件会把 `app_settings.json`、`materials_config.json`、主台账等配置文件也当作待处理文件扫描。

建议在桌面或资料盘建立三个文件夹：

```text
自动归档\      放 contract-router.exe 和 data
待归类文件\    放待整理 PDF、图片、扫描件
整理结果\      放软件生成的归类结果
```

软件界面里：

- `监控文件夹` 选择 `待归类文件`
- `输出主文件夹` 选择 `整理结果`

`监控文件夹` 和 `输出主文件夹` 不能是同一个文件夹。`输出主文件夹` 也不要放在 `监控文件夹` 里面。

## 第一次启动

1. 双击 `contract-router.exe`。
2. 如果 Windows 提示安全警告，确认文件来自本次发放包后再打开。
3. 软件会在 exe 同级生成：

```text
app_settings.json
materials_config.json
```

4. 选择 `监控文件夹`。
5. 选择 `输出主文件夹`。
6. 保持勾选 `预览模式：只生成计划，不复制、不改名`。
7. 点击 `开始监控`。

预览模式不会复制文件，不会改名，也不会移动原始材料。第一次使用先跑预览，看日志确认软件能读到主台账和材料路径。

## 预览模式检查

点击 `开始监控` 后，看下方运行日志。正常情况下会看到类似信息：

```text
开始监控: ...
输出目录: ...
操作清单: ...\_操作日志\operation_manifest_...
当前为预览模式：不会复制、改名或移动文件
发票号清单: ...\data\公司系统出运统计发票号清单.xlsx（若干个发票号）
出运闭环台账: ...\data\出运闭环台账.xlsx（匹配若干个发票号，补充若干个字段，额外若干个发票号）
退税材料规则: ...\materials_config.json
```

如果日志里能显示主台账路径和发票号数量，说明主台账已被读取。

如果没有放 `出运闭环台账.xlsx`，日志会提示未找到出运闭环台账。这不是错误，只表示本次闭环判断只能使用主台账里已有的出运日期、目的国和金额。

预览完成后，点击 `停止监控`。确认没有明显路径错误后，再取消勾选预览模式进行真实整理。

## 真实整理

1. 把本批材料放进 `待归类文件`。
2. 打开软件，选择 `监控文件夹` 和 `输出主文件夹`。
3. 取消勾选 `预览模式：只生成计划，不复制、不改名`。
4. 保持勾选 `生成标准命名副本`。
5. 点击 `开始监控`。
6. 弹出确认框时，确认本批文件数量无误后继续。
7. 等待日志不再新增处理信息。
8. 点击 `停止监控`。

真实整理会把文件复制到输出目录。默认做法是生成标准命名副本，原始材料所在文件夹不移动。

如果本批已有待处理文件超过安全阈值，软件会要求输入：

```text
确认执行
```

只有输入完全一致后才会继续执行。

## 输出结果

输出主文件夹里会生成按发票号或报关号整理后的材料文件夹。材料齐套并且闭环通过时，软件会生成标准命名副本，文件夹名称包含报关号和发票号数字。

常见输出文件包括：

```text
工作日志_YYYY-MM-DD.xlsx
工作日志_YYYY-MM-DD.json
判断报告_YYYY-MM-DD.xlsx
判断报告_YYYY-MM-DD.json
declaration_overrides.json
_操作日志\
```

`工作日志` 记录每个文件的处理结果、源路径、目标路径和说明。

`判断报告` 记录每票材料的齐套情况、缺少材料、重复材料、需要人工确认的文件、闭环状态、闭环证据和冲突信息。

如果放了 `出运闭环台账.xlsx`，报告里还会显示 `闭环字段来源` 和 `台账字段冲突`。`闭环字段来源` 用于确认日期、目的国、金额来自主台账还是补充台账；`台账字段冲突` 用于标记两份台账同一字段不一致的票。

`declaration_overrides.json` 保存手工补录的报关号。

`_操作日志` 保存真实执行或预览时的操作清单。需要撤销上次整理时，软件会读取这里的最新清单。

## 材料状态说明

`判断报告` 里常见状态如下。

`可改名`：必需材料齐全，报关号已识别，闭环校验通过，可以生成标准命名副本。

`已生成改名副本`：已经按报关号和发票号生成标准命名副本，原文件夹保留。

`缺材料`：必需材料没有齐全。查看 `缺少材料` 列，补齐后重新运行。

`待复核`：软件能识别部分信息，但存在需要人工确认的文件或闭环分数不足。查看 `需人工确认文件`、`闭环冲突` 和 `闭环未核验`。

如果 `缺少材料` 为空但状态仍为 `待复核`，通常是闭环证据不足。重点看 `闭环未核验`、`闭环字段来源` 和 `台账字段冲突`。

`待输入报关号`：材料齐了，但没有提取到报关号。软件会提示输入报关号；输入后会记录到 `declaration_overrides.json`。

`不通过`：闭环校验存在冲突。优先查看 `闭环冲突`。

`未找到文件夹`：财务批次里有这个发票号，但输出目录中没有对应材料文件夹。

`不在主台账`：财务批次里的发票号没有在公司主台账中找到。

## 撤销上次整理

真实整理后，如果发现选错输出目录或材料放错批次，可以在停止监控后点击 `撤销上次整理`。

撤销只会按最近一次操作清单删除本软件复制生成的文件。目标文件如果已经被人工修改，软件会跳过，不会强删。

撤销前需要确认：

- 已点击 `停止监控`
- 输出主文件夹仍然是本次整理使用的文件夹
- `_操作日志` 目录还在输出主文件夹里

## 生成财务批次包

先完成材料归类和齐套判断，再生成财务批次包。

操作步骤：

1. 选择本次整理结果所在的 `输出主文件夹`。
2. 点击 `生成财务批次包`。
3. 选择财务给出的发票号 Excel。
4. 等待日志显示处理完成。

财务批次 Excel 需要包含发票号。软件会优先读取表头为 `发票号` 的列；如果没有这个表头，会读取第一列。

生成结果默认放在：

```text
整理结果\财务退税批次包\批次表文件名_YYYYMMDD_HHMMSS\
```

批次包里会生成：

```text
批次统计报告.xlsx
批次统计报告.json
原财务批次表名_已标注.xlsx
```

软件还会按设置回写公司主台账和财务批次表，新增或更新这些字段：

- `退税齐套状态`
- `财务批次`
- `齐套检查时间`
- `缺少材料`
- `闭环状态`
- `闭环分数`
- `退税包路径`
- `备注`

回写 Excel 前，软件会先生成备份文件。处理财务批次前，请关闭主台账和财务批次 Excel，避免文件被占用。

## 常见问题

### 提示未找到公司系统出运统计发票号清单

先检查这三项：

1. `data` 是否和真正双击运行的 `contract-router.exe` 同级。
2. 主台账真实文件名是否为 `公司系统出运统计发票号清单.xlsx`。
3. 是否有旧的 `app_settings.json` 指向以前的路径。

可以直接删除旧的 `app_settings.json`，重新打开 `contract-router.exe`。新版会自动生成默认配置。

如果仍然报错，把报错框里 `请把主台账放到：` 后面的完整路径发给维护人员。那条路径就是软件实际在找的位置。

### 主台账有文件，但提示没有可用发票号

打开主台账，确认第一行有 `发票号` 列，并且下面有实际发票号。

如果发票号在别的行做了合并标题，先整理成普通表格：第一行是表头，第二行开始是数据。

### 缺少材料为空，但状态还是待复核

这类情况通常说明材料齐了，但闭环判断还没拿到足够证据。打开 `判断报告_YYYY-MM-DD.xlsx`，按这几列看：

- `闭环未核验`：日期、目的国、金额是否缺少台账字段或材料证据。
- `闭环字段来源`：字段来自主台账还是 `出运闭环台账.xlsx`。
- `台账字段冲突`：主台账和补充台账是否同一字段不一致。
- `闭环冲突`：材料内容和台账字段是否冲突。

如果主台账只有发票号，没有出运日期、目的国和金额，可以把补充明细另存为：

```text
自动归档\data\出运闭环台账.xlsx
```

再重新点击 `检查退税材料`。

### 开始监控后把配置文件也扫进来了

监控文件夹选错了。不要选择 `自动归档` 或桌面根目录。单独建立 `待归类文件`，只放本批待整理材料。

### 财务批次处理失败，提示 Excel 正在被打开

关闭公司主台账、财务批次表、已标注批次表，再重新执行。

### 有文件进入未匹配发票号

通常是文件名、父文件夹名、OCR 内容里没有能对应主台账的发票号。先检查：

- 文件是否放错批次
- 文件名或上级文件夹是否包含发票号数字
- 主台账是否有对应发票号

采购合同可以不带 `LY` 前缀，只要文件名里的数字能唯一对应主台账发票号。

### 金额、国家或日期冲突

打开 `判断报告_YYYY-MM-DD.xlsx`，查看 `闭环冲突`。常见原因是外销发票金额和主台账金额不同、外销国家和主台账目的国不同、提单日期和主台账出运日期相差过大。

先核对业务资料，再决定是修改台账、替换材料，还是把该票保留为人工复核。

## 日常使用检查清单

每次运行前检查：

- `contract-router.exe` 和 `data` 在同一层。
- `data` 里有 `公司系统出运统计发票号清单.xlsx`。
- 如果要核对日期、目的国和金额，`data` 里有 `出运闭环台账.xlsx`。
- `待归类文件` 里只放本批材料。
- `整理结果` 不在 `待归类文件` 里面。
- 主台账和财务批次表没有被 Excel 打开。

每次运行后检查：

- 运行日志没有主台账路径错误。
- `工作日志_YYYY-MM-DD.xlsx` 已生成。
- `判断报告_YYYY-MM-DD.xlsx` 已生成。
- `缺材料`、`待复核`、`不通过` 的票已人工看过。
- 需要发给财务的批次包在 `财务退税批次包` 目录下。
